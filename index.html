Universal Asynchronous Receiver/Transmitter
UART is a computer hardware device that translates data between parallel and serial forms. The basic idea is to disintegrate a message into smaller chunks at the source, transmit these chunks through the data bus and integrate them again at the destination to generate the original message. It is always fun to code with logic gates, the results are almost immediate. We did optimisations to minimise the processing delay, this included reducing the number of states in the finite state machine and making sure the control flow touches only necessary number of hardware components of the circuit. The check on the control flow part was done by using different logic models representing the same circuit and the model with the minimum latency was given preference in the final architecture. We used Spartan-6 FPGA because of its low risk, low cost and low power consumption attributes. The coding was done in VHDL using the Xilinx platform and Tera-Term was used for testing.

    Autodesk 123D Design iOS
    It is always fun to build stuff from your own hand. I added features which enables the application to do exactly this. Any 3D object around us can be converted into a 3D Mesh Model by taking its snapshots from different angles and feeding them to 123D Catch. Once we have access to the Mesh Model we can either 3D print it or use 123D Design to decompose the model into 2D slices. The application generates a set of instruction of how to assemble this slices on top of each other to recreate the object. (It is generally easy to 3D print slices rather than the whole model). The project finds application in Lego Industry where we can create almost anything around us out of lego using this concept. Apart from Spatial Decomposition of Meshs, I also improved the Algorithmic Library "Timberline" written in C++ of Autodesk which has functions dealing with Mechanics, Physics, Aero-dynamics etc of the 3D Models. A great learning experience indeed :)

    Reed-Solomon Correction
    The project comprises of end-to-end communication system which is able to correct the errors introduced by the medium into the message transmitted. The setup involves two arduino-UNO each at source and destination connected by wire. Each arduino is in turn connected to a laptop which acts as an user interface to input/display the messages transmitted. Messages transmitted across include necessary overhead to detect error in the bits of the message and correct them. The error correction technique used is Reed-Solomon error correction. The software used is written in Octave and C++.
